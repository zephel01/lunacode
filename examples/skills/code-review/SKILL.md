# Code Review スキル

## 概要
コードの品質・セキュリティ・パフォーマンスをレビューし、改善提案を行います。

## レビュー手順

このスキルがアクティブな場合、以下の手順に従ってください:

1. **対象ファイルの読み込み**: `read_file` ツールで対象のソースコードを読み込む
2. **静的解析**: コードを以下の観点で分析する

### チェック項目

**バグ・エラー**
- 未処理の例外やエラーハンドリング漏れ
- null/undefined の参照可能性
- 型の不整合
- 境界値の考慮漏れ

**セキュリティ**
- インジェクション脆弱性（SQL, XSS, コマンド）
- ハードコードされた秘密情報
- 入力値のバリデーション不足
- 不適切なパーミッション

**パフォーマンス**
- 不要なループやネスト
- メモリリーク（イベントリスナーの解除漏れ等）
- N+1 クエリ問題
- 大きなデータの非効率な処理

**可読性・保守性**
- 命名規則の一貫性
- 関数の長さと複雑度
- コメントの適切さ
- DRY 原則の違反

3. **結果の出力**: 以下のフォーマットで結果を `write_file` で出力する

```
## Code Review Results

### Critical (即時修正が必要)
- [ファイル名:行番号] 説明

### Warning (修正を推奨)
- [ファイル名:行番号] 説明

### Info (改善提案)
- [ファイル名:行番号] 説明

### Summary
- 対象ファイル数: N
- Critical: N件
- Warning: N件
- Info: N件
```

## 制約
- レビューは客観的・建設的に行う
- 問題点だけでなく、良い点も1つ以上指摘する
- 修正コードの提案を含める
