# Refactor スキル

## 概要
既存コードのリファクタリングを安全に実施します。動作を変えずにコード品質を向上させます。

## リファクタリング手順

このスキルがアクティブな場合、以下の手順に従ってください:

1. **現状把握**: `read_file` で対象ファイルを読み込み、現在の構造を理解する
2. **問題特定**: リファクタリングが必要な箇所を特定する
3. **計画立案**: 変更計画を Thought で述べる（何を、なぜ、どう変更するか）
4. **段階的修正**: `edit_file` で一つずつ安全に変更を適用する
5. **検証**: 変更後のファイルを `read_file` で確認する

## リファクタリングパターン

### 抽出系
- **関数の抽出**: 長い関数を意味のある小さな関数に分割
- **変数の抽出**: マジックナンバーや複雑な式を名前付き変数に
- **クラスの抽出**: 複数の責務を持つクラスを分割

### 単純化系
- **条件式の単純化**: ネストした if 文をガード節やearly return に
- **ループの単純化**: 手動ループを map/filter/reduce に
- **パラメータオブジェクト**: 多すぎる引数をオブジェクトに

### 命名系
- **変数名の改善**: 意図が明確な名前に変更
- **関数名の改善**: 動詞+名詞の命名規則に統一
- **定数の導入**: ハードコードされた値に名前を付ける

## 制約
- **動作を変えない**: リファクタリングは外部から見た動作を変更してはいけない
- **一度に一つ**: 1回の edit_file で1つの変更のみ行う
- **変更前後の確認**: 変更前にファイルを読み、変更後にも読んで確認する
- **大きな変更は分割**: 大規模なリファクタリングは複数の小さなステップに分ける
